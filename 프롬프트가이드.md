# 프롬프트 가이드 - AI와 함께 개발하기

이 프로젝트를 만들 때 실제로 사용한 프롬프트를 정리했습니다.
비개발자도 AI에게 이런 식으로 요청하면 원하는 결과를 얻을 수 있습니다.

---

## 이 프로젝트가 만들어진 과정

### 1. 아이디어 단계 — "이런 거 가능해?"

> 중학교 수학학원 선생님인데 내가 애들 수학 퀴즈풀게하려고 간단한 웹을 만들려 하거든?
> 애들은 웹에서 문제를 풀면 그 결과가 내 노션이나 구글독스같은곳에 저장되게 하려고해
> 이래야 비용이 하나도 안들거같은데 어 생각은 어떄? 괜찮은 방법이 있니?

**포인트**: 완벽한 기획서가 아니어도 됩니다. "이런 걸 하고 싶다" 정도만 말하면 AI가 방법을 제안해줍니다.

### 2. 우려사항 — "이건 좀 걱정되는데"

> 구글 시트로 하면 애들이 구글계정을 입력해야되잖아
> 애들은 아이패드로 풀거고 다른 설정이 안들었으면 좋겠어

**포인트**: 제약 조건을 솔직하게 말하세요. "학생은 아이패드 사용", "설정 없이 바로", "비용 0원" 같은 조건이 중요합니다.

### 3. 기능 확인 — "이것도 돼?"

> 이차방정식이나, 삼각함수같은거는 도형이 들어갈 수 도 있는데 이거 너가 제안한 방법으로 가능해?

**포인트**: 어려울 것 같은 부분을 미리 물어보세요. AI가 가능한 방법을 알려주거나, 안 되면 대안을 제시합니다.

### 4. 핵심 요구사항 — "내가 원하는 건 이거야"

> 내가 딸깍하면 너가 자동으로 문제 새로 만들어서 업로드 했으면 좋겠고,
> 번거로운 재작업이 없이 일관된 품질을 보장할 수 있으면 좋겠어

**포인트**: "원클릭", "자동", "일관된 품질" 같은 키워드가 AI에게 방향을 잡아줍니다.

### 5. 범위 확정 — 간단하게 답변

> 중학교 1,2,3학년 대상이면 될거같은데

> (객관식? 주관식?) → 둘 다

**포인트**: AI가 물어보면 짧게 답해도 됩니다. 모르겠으면 "너가 추천하는 걸로" 해도 됩니다.

### 6. 문제 발견 — "이거 좀 이상한데?"

> 이미지보면 x가 선이랑 겹쳐있고
> x가 정확히 뭘 말하는건지 불분명한데 이거 개선 가능해?

**포인트**: 스크린샷을 찍어서 보여주고 문제를 설명하면 AI가 바로 고칩니다. 기술 용어를 몰라도 "이거 이상해", "여기 겹쳐" 정도면 충분합니다.

### 7. 배포 — "여기 올려줘"

> https://github.com/dongmin1213/math-quizzes 레포 만들었는데 여기 올려줄 수 있어?

> image폴더는 안올라가도될거같은데..

**포인트**: URL만 알려주면 AI가 알아서 git 작업을 해줍니다.

## 실전 프롬프트 모음

### 새 단원 추가하기

```
중1에 "통계" 단원 생성기를 추가해줘.
기존 생성기 패턴 그대로 따라서 만들어줘.
config.js랑 admin.html에도 등록해줘.

난이도별로:
- 하: 평균, 중앙값 구하기
- 중: 도수분포표에서 평균 구하기
- 상: 분산, 표준편차 구하기
```

### 디자인 수정하기

```
퀴즈 푸는 화면에서 글씨가 너무 작아.
문제 텍스트 크기를 좀 키워주고, 보기 버튼도 더 크게 해줘.
아이패드에서 보기 좋게.
```

```
결과 페이지 색상이 너무 밋밋해.
정답은 초록, 오답은 빨간색으로 더 확실하게 구분되게 해줘.
```

### 기능 추가하기

```
타이머 기능 추가해줘.
- 문제당 2분 제한시간
- 상단에 남은 시간 표시
- 시간 끝나면 자동으로 다음 문제로 넘어가기
- 시간 초과한 문제는 미응답 처리
```

```
학생이 제출 후에 오답만 모아서 다시 풀기 기능을 만들어줘.
result.html에 "오답 다시 풀기" 버튼 추가하고,
틀린 문제만 다시 풀 수 있게 해줘.
```

```
관리자 페이지에 "이전 퀴즈 기록" 기능을 추가해줘.
배포했던 퀴즈를 다시 불러올 수 있게.
```

### 버그 수정 요청하기

```
아이패드 사파리에서 문제 풀다가 화면 꺼졌다 키면 답이 다 날아가.
새로고침해도 답이 유지되게 해줘.
```

```
주관식에서 "1/2" 이랑 "0.5" 를 같은 정답으로 처리해야 하는데
지금은 다르다고 나와. 고쳐줘.
```

### 스타일 변경하기

```
전체적으로 색상 테마를 바꾸고 싶어.
파란색 대신 보라색 계열로 바꿔줘.
```

```
학생 입장 페이지에 학원 로고 이미지를 넣고 싶어.
상단에 로고 넣을 수 있게 해줘.
```

### 데이터/결과 관련

```
관리자 페이지에서 학생별 성적 추이를 그래프로 볼 수 있게 해줘.
여러 번 퀴즈 본 학생의 점수 변화를 차트로 보여줘.
```

```
결과를 CSV로 다운로드 받을 수 있게 해줘.
관리자 페이지에 "엑셀 다운로드" 버튼 추가.
```

## 좋은 프롬프트 작성법

### 1. 배경 + 원하는 것 + 조건

```
[배경] 지금 학생들이 아이패드로 퀴즈를 푸는데
[원하는 것] 문제 풀 때 메모할 수 있는 공간이 있으면 좋겠어.
[조건] 터치로 손글씨 쓸 수 있게 해줘. 제출할 때 메모는 저장 안 해도 돼.
```

### 2. "기존 패턴 따라서" 를 꼭 넣기

AI가 프로젝트의 기존 코드와 다른 스타일로 만들지 않도록:

```
✅ 좋은 예: "기존 생성기 패턴 따라서 새 단원 추가해줘"
❌ 나쁜 예: "새 단원 추가해줘" (→ 다른 스타일로 만들 수 있음)
```

### 3. 스크린샷 활용하기

글로 설명하기 어려운 건 **스크린샷**이 최고입니다:

```
(스크린샷 첨부)
여기 보면 수식이 깨져서 나오는데 고쳐줘.
```

```
(스크린샷 첨부)
이 부분 레이아웃이 아이패드에서 이상하게 나와.
버튼이 화면 밖으로 나가.
```

### 4. 한 번에 하나씩

```
✅ 좋은 예: "타이머 기능 추가해줘"
   → 완성 후: "타이머 디자인을 좀 더 크게 해줘"

❌ 나쁜 예: "타이머 추가하고 오답노트도 만들고 성적 그래프도 넣고 CSV 다운로드도 해줘"
   → 한꺼번에 너무 많으면 품질이 떨어질 수 있음
```

### 5. 결과 확인 요청하기

```
수정한 거 로컬 서버 띄워서 테스트해줘.
스크린샷으로 보여줘.
```

## Claude Code 시작하기

### 처음 사용할 때

1. 프로젝트 폴더에서 터미널 열기
2. `claude` 입력
3. 대화하듯이 요청하면 됩니다

```bash
cd math-quizzes
claude
```

### 첫 대화 예시

```
이 프로젝트는 중학교 수학 퀴즈 웹앱이야.
CLAUDE.md 읽어보고 프로젝트 구조 파악해줘.
그 다음에 내가 요청하는 거 수정해줄게.
```

> Claude Code는 `CLAUDE.md`를 자동으로 읽기 때문에 위 설명은 생략해도 됩니다.
> 바로 원하는 작업을 요청해도 괜찮습니다.

### 작업 흐름

```
1. 요청하기     → "타이머 기능 추가해줘"
2. 코드 수정됨  → AI가 파일 수정
3. 확인하기     → "로컬에서 테스트해봐" 또는 직접 브라우저에서 확인
4. 피드백       → "타이머 위치를 오른쪽으로 옮겨줘"
5. 완료되면     → "커밋하고 깃허브에 푸시해줘"
```
